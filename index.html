ExtIPs

Extitute Improvement Proposals (ExtIPs) describe standards for the Ethereum platform, including core protocol specifications, client APIs, and contract standards. Network upgrades are discussed separately in the Ethereum Project Management repository.

Contributing

First review ExtIP-1. Then clone the repository and add your ExtIP to it. There is a template ExtIP here. Then submit a Pull Request to the Extitute's ExtIPs repository.

ExtIP status terms

  • Idea - An idea that is pre-draft. This is not tracked within the ExtIP Repository.
  • Draft - The first formally tracked stage of an ExtIP in development. An ExtIP is merged by an ExtIP Editor into the ExtIP repository when properly formatted.
  • Review - An ExtIP Author marks an ExtIP as ready for and requesting Peer Review.
  • Last Call - This is the final review window for an ExtIP before moving to FINAL. An ExtIP editor will assign Last Call status and set a review end date (`last-call-deadline`), typically 14 days later. If this period results in necessary normative changes it will revert the ExtIP to Review.
  • Final - This ExtIP represents the final standard. A Final ExtIP exists in a state of finality and should only be updated to correct errata and add non-normative clarifications.
  • Stagnant - Any ExtIP in Draft or Review if inactive for a period of 6 months or greater is moved to Stagnant. An ExtIP may be resurrected from this state by Authors or ExtIP Editors through moving it back to Draft.
  • Withdrawn - The ExtIP Author(s) have withdrawn the proposed ExtIP. This state has finality and can no longer be resurrected using this ExtIP number. If the idea is pursued at later date it is considered a new proposal.
  • Living - A special status for ExtIPs that are designed to be continually updated and not reach a state of finality. This includes most notably ExtIP-1.

ExtIP Types

ExtIPs are separated into a number of types, and each has its own list of ExtIPs.

Meta ({{site.pages|where:"type","Meta"|size}})

Describes a process surrounding Extitute or proposes a change to (or an event in) a process. Process ExtIPs are like Standards Track ExtIPs but apply to areas other than the Ethereum protocol itself.

Governance ({{site.pages|where:"type","Governance"|size}})

Governance ExtIPs describe any change that affects the core governance processes of the Extitute. Examples include a change to the governance protocol, procedures, guidelines, or decision-making process, or a change in the constitutional principles and values of the Extitute.

Operational ({{site.pages|where:"type","Operational"|size}})

Operational ExtIPs describes any change that affects the core operational processes of the Extitute. Examples include the addition of a new qualification badge, the introduction of a new sphere of activity, and any change or addition that affects the operations and/or the missions of the Extitute.

Informational ({{site.pages|where:"type","Informational"|size}})

Informational ExtIPs describes an extitutional design issue, or provides general guidelines or information to the Extitute, but does not propose a new feature.

Qualification ({{site.pages|where:"type","Qualification"|size}})

Qualification ExtIPs describe a qualification & qualification issuer type & requirements.
